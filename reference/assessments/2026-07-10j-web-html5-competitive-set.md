## What the browser HTML5 run-and-gun tier actually looks like (observed)
GUN N' RUN (a top-listed itch HTML5 run-and-gun): **minimalist, stark, near-monochrome + pink
pixel art** — small blocky characters (a face on a box), dark near-black backgrounds, girder/
truss platforms, big pink muzzle blasts, pink-coded projectiles, an ammo-pip HUD. Clean and
readable, but **low-detail / jam-quality** production — no rich environment, no detailed
sprites, no parallax depth.

## Positioning vs OUR game (the literal browser win condition)
This is the tier a browser player actually chooses between. **OUR game clearly EXCEEDS it on
visual fidelity + content:** OURS has real detailed 16-bit hero/enemy sprites (idle/run/leap/
prone), a tiled night-jungle environment with a 5-layer parallax + moon/stars/fireflies, the
full arcade M/S/L/F weapon roster, three enemy types + a two-phase boss, hit-stop/telegraphs,
and clean touch controls — vs GUN N' RUN's stark mono+pink jam aesthetic. On the DIRECT
browser competitive axis, **OURS is above the typical itch/CrazyGames HTML5 run-and-gun.**

**Caveat (honest):** GUN N' RUN is ONE sample; the itch HTML5 run-and-gun tag spans jam
entries of widely varying quality (a few are more polished). The tier *skews* minimalist, so
"OURS exceeds the typical one" is well-supported, but this is a sample, not an exhaustive tier
survey. If a specific higher-fidelity HTML5 competitor is named, capture + compare it directly.
